## ADDED Requirements

### Requirement: 主题切换与持久化
系统SHALL为前端提供可切换的亮色与暗色主题，并在后续访问中保持用户选择。

#### Scenario: 用户切换主题
- **GIVEN** 用户位于任意支持主题的页面
- **WHEN** 用户触发主题切换入口
- **THEN** 系统在当前页面即时应用目标主题
- **AND** 不要求用户刷新页面或重新登录

#### Scenario: 刷新后保持主题
- **GIVEN** 用户已在当前设备选择主题
- **WHEN** 用户刷新页面或重新打开站点
- **THEN** 系统恢复用户上次选择的主题
- **AND** 核心壳层与基础控件使用相同主题

### Requirement: 语义化主题令牌
系统SHALL使用统一的语义化设计 token 驱动背景、前景、表面、边框、输入态、焦点环和状态反馈样式。

#### Scenario: 基础控件共享同一组主题语义
- **GIVEN** 页面同时渲染按钮、输入框、卡片和弹窗
- **WHEN** 系统应用亮色或暗色主题
- **THEN** 这些控件使用同一组语义化颜色和表面层级
- **AND** 不依赖页面内单独定义的硬编码色值来维持可读性

#### Scenario: 主题切换覆盖状态样式
- **GIVEN** 用户在表单与操作区进行悬停、聚焦、禁用或错误态交互
- **WHEN** 主题发生切换
- **THEN** 系统保留各状态的可辨识层级
- **AND** 焦点与反馈样式在两种主题下均保持可见
