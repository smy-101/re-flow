# 时间戳显示修复规格文档

本变更为纯前端 bug 修复，不引入新能力，不修改现有 API 或数据结构的规范。

## Capabilities 概述

根据 `proposal.md`，本变更不涉及新的能力定义，也不修改现有能力的需求级行为。所有修改仅限于前端组件的实现细节（时间戳格式转换）。

## 无规格变更

本变更无需创建或修改任何规格文件，原因如下：

1. **无新能力**：不引入新的系统功能
2. **无需求变更**：不修改 API 响应格式、数据库 schema 或用户交互行为
3. **纯实现修复**：仅修正前端时间显示逻辑，不改变系统级契约

## 实现细节

修复方案在 `design.md` 中详细描述，核心修改为：

- 在前端组件中将秒级 Unix 时间戳转换为毫级（乘以 1000）
- 涉及组件：`FeedCard.tsx`、`ItemCard.tsx`、`CraftTemplateCard.tsx`

## 验证方式

修复完成后，验证以下页面的时间显示：
- `/feeds` - 订阅卡片的"最后更新"时间
- `/items` - 文章列表的发布时间
- `/settings/craft` - Craft 模板卡片的"创建于"时间

预期结果：所有时间显示为正确的当前时间（如 2026-03-09），而非 1970-01-21。
