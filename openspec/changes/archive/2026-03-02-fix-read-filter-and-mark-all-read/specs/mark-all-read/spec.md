## ADDED Requirements

### Requirement: 批量标记所有未读文章为已读

系统 SHALL 提供批量标记所有订阅中所有未读文章为已读的功能。

#### Scenario: 在未读页面标记所有文章为已读
- **GIVEN** 用户在 `/items/unread` 页面
- **AND** 用户有未读文章
- **WHEN** 用户点击"全部标记为已读"按钮
- **THEN** 系统显示确认对话框
- **AND** 对话框显示"共 N 篇未读文章将被标记为已读"
- **AND** 用户确认后，系统标记所有未读文章为已读
- **AND** 显示"暂无未读文章"空状态

#### Scenario: 标记成功后刷新列表
- **GIVEN** 用户已标记所有未读文章为已读
- **WHEN** API 操作完成
- **THEN** 系统自动刷新文章列表
- **AND** 列表显示空状态"暂无未读文章"
- **AND** 所有订阅的未读计数更新为 0

#### Scenario: 取消标记操作
- **GIVEN** 用户打开确认对话框
- **WHEN** 用户点击"取消"按钮
- **THEN** 系统关闭对话框
- **AND** 不执行任何标记操作
- **AND** 未读文章状态保持不变

---

### Requirement: 批量标记特定订阅的未读文章为已读

系统 SHALL 提供批量标记特定订阅中所有未读文章为已读的功能。

#### Scenario: 在订阅详情页标记该订阅的所有未读文章
- **GIVEN** 用户在 `/feeds/[feedId]` 页面
- **AND** 当前过滤器为"未读"
- **AND** 该订阅有未读文章
- **WHEN** 用户点击"全部标记为已读"按钮
- **THEN** 系统显示确认对话框
- **AND** 对话框显示"本订阅共 N 篇未读文章将被标记为已读"
- **AND** 用户确认后，系统仅标记该订阅的未读文章为已读
- **AND** 该订阅显示"0 篇未读"

#### Scenario: 在"已读"过滤器下不显示标记按钮
- **GIVEN** 用户在 `/feeds/[feedId]` 页面
- **WHEN** 用户切换到"已读"过滤器
- **THEN** 系统隐藏"全部标记为已读"按钮

#### Scenario: 在"全部"过滤器下不显示标记按钮
- **GIVEN** 用户在 `/feeds/[feedId]` 页面
- **WHEN** 用户切换到"全部"过滤器
- **THEN** 系统隐藏"全部标记为已读"按钮

---

### Requirement: 标记操作确认对话框

系统 SHALL 在执行批量标记操作前显示确认对话框。

#### Scenario: 显示确认对话框
- **GIVEN** 用户点击"全部标记为已读"按钮
- **WHEN** 对话框打开
- **THEN** 系统显示操作说明
- **AND** 显示将受影响的未读文章数量
- **AND** 说明此操作不会删除文章
- **AND** 提供"确认"和"取消"两个按钮

#### Scenario: 关闭确认对话框
- **GIVEN** 确认对话框处于打开状态
- **WHEN** 用户点击关闭按钮、点击遮罩层或按 ESC 键
- **THEN** 系统关闭对话框
- **AND** 不执行任何标记操作

---

### Requirement: 标记操作加载状态

系统 SHALL 在标记操作执行期间显示加载状态。

#### Scenario: 按钮显示加载状态
- **GIVEN** 用户在确认对话框中点击"确认"
- **WHEN** API 请求正在执行
- **THEN** 系统禁用"确认"按钮
- **AND** 按钮显示加载指示器
- **AND** 防止重复点击

#### Scenario: 操作完成后恢复状态
- **GIVEN** API 请求已完成
- **WHEN** 标记操作成功
- **THEN** 系统关闭确认对话框
- **AND** 刷新文章列表
- **AND** 按钮恢复可点击状态

---

### Requirement: API 响应处理

系统 SHALL 正确处理批量标记 API 的响应。

#### Scenario: 成功响应处理
- **GIVEN** 用户确认标记操作
- **WHEN** API 返回成功响应 `{ success: true, count: N }`
- **THEN** 系统更新本地状态
- **AND** 刷新文章列表
- **AND** 关闭确认对话框

#### Scenario: 错误响应处理
- **GIVEN** 用户确认标记操作
- **WHEN** API 返回错误响应
- **THEN** 系统显示错误提示
- **AND** 保持确认对话框打开状态
- **AND** 允许用户重试或取消
