## Manual Test Cases

### 2.1 主题系统

**Given** 用户已登录并访问 dashboard 或认证页面
**When** 用户通过顶栏主题按钮或用户菜单切换浅色、深色、跟随系统，并刷新页面或重新进入站点
**Then** 页面保持用户上次选择的主题
**And** 顶栏、侧栏、卡片、表单和反馈提示在两种主题下都保持足够对比度与可读性

### 2.2 基础控件

**Given** 页面中存在按钮、输入框、选择器、弹窗、抽屉、菜单和 Toast
**When** 用户触发 hover、focus、disabled、error、loading 等状态
**Then** Button、Input、Select、Dialog、Sheet、Dropdown Menu 和 Toast 使用统一的圆角、边框、焦点环和反馈颜色
**And** 状态变化在浅色与深色主题下都可辨识

### 2.3 Dashboard 壳层

**Given** 用户在桌面端与移动端访问受保护页面
**When** 用户打开侧栏、抽屉、用户菜单并切换页面与主题
**Then** 顶栏、侧栏、抽屉、搜索占位区和主题入口保持统一结构
**And** 当前导航高亮、收藏计数与用户菜单交互在两端行为一致

### 2.4 认证壳层

**Given** 用户访问登录、注册、忘记密码和重置密码页面
**When** 用户输入字段、触发校验错误、发送验证码并提交表单
**Then** 页面共享相同的容器、标题层级、按钮样式和错误/成功反馈
**And** 验证码按钮倒计时、表单禁用态与跳转入口保持一致

### 2.5 高频工作流

**Given** 用户使用添加订阅、AI 配置、订阅卡片和文章列表
**When** 用户执行验证、提交、刷新、筛选和状态切换
**Then** 表单字段、说明文本、卡片层级、空状态、错误态和 Toast 使用统一样式
**And** 订阅刷新成功/失败、文章列表筛选、AI 配置测试结果的反馈位置和语义保持一致
