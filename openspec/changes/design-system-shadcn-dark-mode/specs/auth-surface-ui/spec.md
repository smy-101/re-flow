## ADDED Requirements

### Requirement: 认证共享壳层
系统SHALL为登录、注册、忘记密码和重置密码页面提供一致的认证壳层。

#### Scenario: 访问任一认证页面
- **GIVEN** 用户访问 `/login`、`/register`、`/forgot-password` 或 `/reset-password`
- **WHEN** 页面完成渲染
- **THEN** 系统展示统一的认证容器、标题层级和辅助说明区域
- **AND** 页面之间共享一致的视觉节奏与表面样式

### Requirement: 认证表单状态一致
系统SHALL为认证流程中的字段、按钮和状态提示提供一致的交互反馈。

#### Scenario: 字段校验失败
- **GIVEN** 用户在认证表单输入无效内容或遗漏必填字段
- **WHEN** 系统进行校验
- **THEN** 系统以统一的错误样式展示字段状态与说明
- **AND** 按钮和表单布局保持稳定，不因报错造成结构错位

#### Scenario: 提交与发送验证码反馈
- **GIVEN** 用户提交认证表单或发送验证码
- **WHEN** 请求处于加载、成功或失败状态
- **THEN** 系统使用统一的按钮加载态与反馈提示组件
- **AND** 相同状态在不同认证页面表现一致

### Requirement: 认证页面支持亮暗主题
系统SHALL确保认证页在亮色与暗色模式下保持一致的可读性和品牌氛围。

#### Scenario: 认证页切换主题
- **GIVEN** 用户位于任一认证页面
- **WHEN** 系统应用亮色或暗色主题
- **THEN** 认证容器、输入控件、辅助文字和反馈组件同步切换
- **AND** 主要操作按钮与错误提示始终保持足够对比度
